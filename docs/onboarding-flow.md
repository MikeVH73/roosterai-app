# RoosterAI Onboarding Flow

## Goal
Help a new customer reach the first successful schedule creation as quickly and clearly as possible.

## Current dependency chain
Before a user can create a schedule, the following setup steps must be completed:

1. Functions & Skills
2. Locations
3. Location Types
4. Departments
5. Employees
6. Contract Hours
7. Availability
8. Schedule creation
9. Optional: use Planning Helper

## Main risk
New users may not understand that these steps depend on each other.
If they do not receive guidance, they may stop before creating their first schedule.

## Desired onboarding outcome
The user should be guided step by step toward:
- understanding the setup order
- completing the minimum required data
- generating the first schedule successfully

## Ideal onboarding support
RoosterAI should eventually provide:
- a setup wizard
- progress tracking
- contextual help per step
- proactive onboarding guidance
- support escalation when the user gets stuck

## Activation moment
The key activation moment is:
**the first successfully generated schedule**

## Notes
The onboarding experience should be optimized for Dutch-speaking users with limited patience and minimal training.
