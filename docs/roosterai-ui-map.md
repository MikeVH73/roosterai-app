# RoosterAI UI Map

This document describes the main areas of the RoosterAI user interface and their purpose.

---

# Main Sections

## Dashboard
Purpose:
Provide overview of planning status and quick access to important actions.

Possible elements:
- Planning status
- Alerts
- Quick actions
- Recent schedules

---

## Employees
Manage all employees that can be scheduled.

Key elements:
- Employee profiles
- Skills / functions
- Contract hours
- Availability
- Departments

---

## Locations
Defines where employees work.

Includes:
- Locations
- Location types
- Departments per location

---

## Functions & Skills
Defines what employees are capable of doing.

Used for:
- assigning correct staff to shifts
- filtering employees during planning

---

## Availability
Defines when employees are available to work.

Examples:
- fixed weekly availability
- vacation
- sickness
- temporary blocks

---

## Schedule
Core feature of RoosterAI.

Capabilities:
- create schedules
- edit schedules
- view coverage
- detect conflicts

---

## Planning Helper
AI assisted planning feature.

Purpose:
Help generate or optimize schedules automatically.

Potential capabilities:
- suggest shift allocation
- detect conflicts
- improve coverage
- balance workload

---

## Settings
System configuration for the organisation.

Possible items:
- company settings
- planning rules
- notifications
